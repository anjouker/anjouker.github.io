汇编语言读书笔记

  • 汇编
  • 笔记

1.汇编语言 有3类指令组成

  • 汇编指令：机器码的助记符，有对应的机器码
  • 伪指令：没有对应的额机器码，由编译器执行，计算机并不执行。
  • 其他符号：如+、-、*、/等，由编译器识别，没有对应的机器码。

核心为汇编指令，它决定了汇编语言的特性。

2.指令和数据没有本质上的区别，只是应用上的概念而已。

例如内存中的二进制信息 1000100111011000 ，有以下两种看法。

1000100111011000 —> 89D8H（数据）

1000100111011000 —> mov ax,bx （程序）

3.CPU想要对数据进行读写，必须与外部器件（芯片）进行3类信息交互。

  • 存储单元的地址。（地址信息）
  • 器件的选择，读或写的命令，（控制信息）
  • 读或写的数据。（数据信息）

总线从物理上说，就是一根根导线的集合；从逻辑上讲，可分为3类：地址总线，控制总线，数据总线。

4.地址总线

一个CPU有N根地址线，就说这个CPU的地址总线的宽度为N。最多可以寻找2的N次方个内存单元。

5.数据总线

数据总线的宽度决定了CPU与外界的数据传输速度。
8086有16根地址总线，就可以一次性传输16未数据。

6.控制总线

控制总线的宽度决定了CPU对外部器件的控制能力。

7.内存地址空间

CPU的地址总线的宽度为N，那么其可以寻址2的N次方个内存单元。这2的N次方个内存单元就构成这个CPU的内存地址空间。
包括但不限于：

  • RAM（主存储器）
  • ROM（装有系统BIOS）
  • RAM（显存）
  • ROM（装有显卡BIOS）

  • ROM（装有网卡BIOS）

    这些存储器在物理上是独立的，但是有两个共同点：

  • 都和CPU的总线相连
  • CPU对它们进行读或写的时候都是通过控制总线发出内存读写命令

8.通用寄存器

  • 包括AX、BX、CX、DX四个寄存器，在8086CPU中为16位的，存放两个字节。每个寄存器均可分为两个独立使用的8位寄存器。

9.几条汇编指令

mov指令

mov ax,18

mov ah,78

mov ax,bx

……

！注意：在写一条汇编指令或者一个寄存器的名称时不区分大小写。

注意：在进行数据的传送或者运算的时候，指令的两个操作对象的位数应该是一致的。

下列指令是错误的：

mov ax,bl

mov bh,ax

mov al,20000 （8位寄存器的最大可存放值为255）

add指令

add ax.8

add ax,bx

注意：当单独使用AL和AH寄存器时，CPU会认为这是两个不相关的寄存器。所以，不要错误的以为，add al,93H这样的指令产生的进位会存储在ah中。

10.16位结构的CPU

16位CPU一般来说包括下面几方面的特性：

  • 运算器一次最多处理16位的数据
  • 寄存器的最大宽度为16位
  • 寄存器和运算器之间的通路为16位

8086CPU是16位的，这也就是说，在8086内部，能够一次性处理、传输、暂时存储的信息的最大长度是16位的。

而内存单元的地址在送上地址总线之前，必须在CPU中处理、传输、暂时存放，对于16位CPU，能一次性处理、传输、暂时存储16位 的地址。

推而广之，32位，64位的CPU也是如此。

11.8086CPU寻址方法

8086CPU有20位 地址总线，可以传送20位地址，达到1MB的寻址能力。

而8086CPU有是16位结构，从内部结构看，如果地址简单的从内部发出，那么它只能送出16位 的地址，表现出的寻址能力为64KB。

8086CPU采用一种在内部用两个16位地址合成的方法形成一个20位的物理地址。

CPU通过“物理地址=段地址*16+偏移地址”的方法合成物理地址。

12.“物理地址=段地址*16+偏移地址”的本质

因为CPU只有16位这一限制条件，导致其寻址能力只有64KB，但是我们希望能使用大的内存，所以通过把一个20位的地址分开成为两部分来存储，需要使用的时候再通过CPU的计算得出。虽然牺牲一些运算效率，但是也得到了更大的可用内存。

13.段

段只是一个逻辑概念，而不是真实存在的。

在编程时，我们可以根据需要，将若干地址连续的内存单元看作是一个段。

  • 段的起始地址必然是16的倍数。
  • 段的最大长度为64KB。

CPU可以用不同的段地址和偏移地址合成同一个物理地址。

for example

21F60H -> 2000H:1F60H
-> 2100H:0F60H
-> 21F0H:0060H ……

14.段寄存器

段寄存器包括四个，CS、DS、SS、ES

15.CS和IP

CS和IP是8086CPU中最关键的寄存器，它们指示了当前CPU要读取的指令的物理地址。

在8086PC机中，任意时刻，CPu将CS:IP指向的内容当作指令执行。

也就是说CPU一直在重复下列步骤：

  • 从CS:IP指向的内存单元读取指令，读取的指令进入指令缓冲器。
  • IP=IP+所读的指令长度，从而指向下一条指令。
  • 执行指令。

在8086CPU加电启动或者复位后，CS和IP被设置为CS=FFFFH，IP=0000H。

16.在CPU中，程序员可以用指令读写的部件只有寄存器，程序员可以通过改变寄存器的内容实现对CPU的控制。

17.改变CS、IP的指令

21.栈

任意时刻，SS：SP指向栈顶元素
push ax的执行，分以下两步完成：
1）SP=SP-2，SS：SP指向当前栈顶前面的单元，以当前栈顶前面的单元为新的栈顶
2）将ax中的内容送入SS：SP指向的内存单元，SS：SP指向新的栈顶
pop ax的执行刚好相反：
1）将SS：SP指向的内存单元的数据送入ax中
2）SP=SP+2，SS：SP指向当前栈顶下面的单元，以当前栈顶下面的单元为新的栈顶
8086CPU不保证我们对栈的操作不会越界
以下指令也是正确的
push 寄存器
pop 寄存器
push 段寄存器
pop 段寄存器

只要在【……】中使用了寄存器bp，而指令中没有显性的给出段地址，则段地址默认在ss中

27.操作内存单元长度

在没有给出寄存器名称的时候，用操作符X ptr 指明内存单元的长度是很有必要的。

28.div指令

1）除数，有8位和16位两种，在寄存器中或者内存单元中
2）被除数，默认在AX或者DX和AX中，如果除数为8位，被除数为16位，默认在AX中存放，如果除数为16位，被除数则为32为，在DX和AX中存放，DX存放高16位，AX存放低16位。
3）结果：如果除数为8位，则AL存储出发操作的商，AH存储出发操作的余数；如果除数为16位，则AX储存除法操作的商，DX存储余数。

29.定义数据伪指令

db，一个字节，01H
dw，一个字，0001H
dd，2个字，00000001H
duo，定义重复的数据

30.操作符offset

操作符offset在汇编语言中是由编译器处理的符号，它的功能是取得标号的偏移地址。

31.jmp指令

jmp short 标号
这样的jmp指令实现的是段内转移，它对IP的修改范围为-128~127.

CPU在执行的时候并不是转移的目的地址。

jmp far ptr 标号
实现的是段间转移，又称为远转移

jmp 16位 reg
功能 （IP）=（16位 reg）

jmp word ptr 内存单元地址（段内转移）
jmp dword ptr’ 内存单元地址（段间转移）
（CS）=（内存单元地址+2）
（IP）=（内存单元地址）

32.jcxz指令

jcxz指令为由条件转移指令，所有的有条件的转移指令都是短转移，在对应的机器码中只包含转移的位移，而不是目标地址，对IP范围为 -128~12

33.ret和retf

ret指令用栈中的数据，修改IP的内容，从而实现近转移
retf指令用栈中的数据，修改CS和IP的内容，从而实现远转移
CPU执行ret指令时，进行下列2步操作
(IP)=((ss)*16+(sp))
(sp)=(sp)+2
CPU执行retf指令时，进行下列4步操作
(IP)=(ss)*16+(sp))
(sp)=(sp)+2
(CS)=((ss)*16+(sp))
(sp)=(sp)+2

将当前的IP或者CS和IP压入栈
转移

35.mul指令

1）两个相乘的数，要么都是8位的，要么都是16位的，如果是8位的，一个默认放在AL’中，另一个放在8位reg或者内存单元中，如果是16位，一个默认在AX中，另一个放在16位reg或内存单元中。
2）结果：如果是8位乘法，结果默认在AX中，如果是16位乘法，结果高位放在DX中存放，低位在AX中存放

36.标志寄存器

1）ZF标志
flag的第6位是ZF，零标志位，其结果是否是0，如果是0，那么ZF=1，如果不为0，那么ZF=0.
2）PF标志
flag的第2位，奇偶标志位，他记录相关指令执行后，其结果的所有bit中1的个数是否为偶数，如果1的个数为偶数，PF=1，如果为奇数，那么PF=0.
3）SF标志
flag的第7位是SF，符号标志位，他记录相关指令执行之后，其结果是否为负，如果结果为负，
4)CF标志
flag的第0位是CF，进位标志位，一般情况下，在进行无符号运算时，它记录了运算结果的最高有效位向更高位的进位值，或从更高位的借位值。
5）OF标志
溢出标志，只是对有符号数而言的。溢出时OF=1，没有溢出时OF=0.

37.adc指令

adc是进位加法指令，它利用了CF位上的进位值。
asc 操作对象1，操作对象2
功能：操作对象1=操作对象1+操作对象2+CF

38.sbb指令

sbb是带借位减法指令，它利用了CF位上的进位值
sbb 操作对象1，操作对象2
功能：操作对象1=操作对象1-操作对象2-CF

39.cmp指令

cmp是比较指令，cmp的功能相当于减法指令，只是不保存结果，cmp指令执行后，只对寄存器产生影响

40.条件转移指令

je 等于则转移 ZF=1
jne 不等于则转移 ZF=0
jb 低于则转移 CF=1
jnb 不低于则转移 CF=0
ja 高于则转移 CF=0且ZF=0
jna 不高于则转移 CF=1且ZF=1

41.DF标志和串传送指令

flag的第10位是DF，方向标志位。
DF=0，每次操作后SI，DI递增
DF=1，每次操作后SI，DI递减
movsb
功能：mov es:[di],byte ptr ds:[si]
if df=0
inc si
inc di
if df=1
dec si
dec di

42.pushf和popf

pushf是将标志寄存器压栈，而popf相反

43.iret指令

相当于
pop IP
pop CS
popf

44.装载0号中断代码

编写可以显示“overflow！”的中断处理程序：do0
将do0送入内存0000：0200
将do0的入口地址0000：0200存储在中断向量表0号表项中

45.shl和shr指令

shl是逻辑左移指令，功能：
将一个寄存器或内存单元中的数据向左移位；
将最后一位写入CF中；
最低位用0补充。

shr是逻辑右移指令，功能：
将一个寄存器或内存单元中的数据向右移位；
将最后移出的一位写入CF中；
最高位用0补充。
